How it works

A deterministic + AI pipeline for UFGS tailoring.

SpecStage runs each UFGS section through a six-stage pipeline. Deterministic rules handle the mechanical decisions; AI handles the project-specific judgment calls. Every decision is traceable.

1

Parse

Each .sec file is parsed into a tree of structural nodes (sections, parts, paragraphs, brackets, fill-ins, references, submittals). The parser preserves the SpecsIntact XML faithfully so the output round-trips.

2

Pass A — Paragraph and option pruning

Deterministic rules prune branch-specific options (USACE / NAVFAC / AFCEC), unit-system options (English / metric), and removed sections. Specifier notes are stripped. Tables are pruned row-by-row against project applicability.

3

Pass B — Brackets and fill-ins

Every bracketed choice [Option A][Option B] and every blank [_____] is resolved by Claude against your project context — branch, location, mission criticality, scope documents, and the surrounding spec text. Each decision returns a confidence score and a written rationale.

4

Pass C — Host nation injection

For OCONUS projects where host nation standards are accepted, equivalent local standards (JIS, IEC, DIN, BS EN, KS, …) are injected alongside US standards in both the REFERENCES article and inline citations.

5

Validate

Output is validated against the UFGS schema. Unresolved blanks, broken references, and orphaned submittal items are flagged for PE review. Nothing leaves the pipeline that wouldn't pass a spec review.

6

Export

Tailored sections export as SpecsIntact-compatible .sec files, ready to drop into your existing {SpecsIntactWD}\Jobs\<project>\pulldata\ folder. Decisions are logged alongside the output for full audit traceability.

Why a hybrid pipeline

Determinism where it works. AI where it matters.

Branch filtering and unit selection have correct, mechanical answers — those run as code. Bracket selections and fill-ins require reading the project context — those run through Claude. The split keeps results reproducible where they should be reproducible, and judgment-aware where they need to be judgment-aware.
